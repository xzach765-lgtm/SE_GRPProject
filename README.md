## Project Description
Our main idea in this project is to develop a practical academic system designed to support both students and lecturers. The system aims to improve communication, acessibility of learning materials, and assignment management all in one single platform. First of all, the system will allow students to have access to all learning materials such as lecture slides and tutorials, submiting their assignments through a fast and secure platform, able to provide lecturer feedback and message them directly through a safe and private messaging system and book a consultation apointment directly from them. As for lecturers, the system allows them to upload teaching materials, create and manage assignments, review student's assignments and provide feedback effectively.

By integrating these acedemic core functions into one single platform, the system enhances productivity and learning effectiveness for both student and lecturers while leaving focused only on core features defined in the project scope.

## Overview of SDLC Work

- Planning
  - We, as a team, gathered stakeholder's requirement by using a Google form that we created. Our team would have to identify the votes of the majority and differentiate which is Must have, Should have, Could have and Won't have. After collecting the responses, we analysed the features suggested by students and lecturers to understand what functionalities are most crucial for the educational application. The team then categorised the requirements into Functional Requirement and Non-Requirement. This planning process helps the team clarify identify the system scope, priotising vital features and ensure that the development process focuses on the most essential requirements first
- Analysis
  - After analysing the data, we came to a conclusion that the highest priority of functions that students want are the ability to view lecture slides and submit homework. For lecturers, it would be the software making them able to post lecture and tutorial slides for use at their students own discretion. Some following important features would be students being able to give tutor feedback and lecturers being able to check the location of the student taking the attendance, either on campus or elsewhere as well as the function of showing the total percentage of people attending class. Other features would include a consultation feature for weaker students to seek help, a feature for students and lecturers to message each other on and a way for students to immediately submit their own homework without needing to do it with another file. Some features that we won't have for now would be students being able to set up a study buddy, facial login to quicken the process of getting into the app and the ability to customise the UI of the app according to their own preferences
- Design
  - For our wireframe, we had a simple layout. We added a frame for the login page, complete with the options of login in as staff or student as well as being able to login with a Google or Apple account. For the register page, students and lecturers can register a new account or click the "Got an account button?" to be redirected to the login page if they ended up on the register page by accident. As for our home pages, the student version will have a button for announcements, direct messages to lecturers, entering attendance and a display classes dropdown menu to direct them to the student's class pages. the lecturer version will have a button for announcements, reviewing direct messages from students, viewing received feedback, and a display classes dropdown menu to direct them to the lecturer's class pages. For the student's class pages, it will have the subject name on top, submit homework button, feedback button, request consultation button, and a view lecture slide and downloadables button. The submit homework button will redirect the student to the assignment questions page, where they can answer the questions. For the lecturer's class pages, it will have the subject name on top, review attendance button, review feedback button and a post lecture slide and downloadables button.
- Prototype Development
  - We have develop a prototype with a basic design as well as basic transitions and animations. For our design, we decided to include the following features for the first sprint. User login for students and lecturer, registering a new account for students and lecturers, a home page for both students and lecturers, students and lecturers being in or in charge of a class, lecture slides beign viewable, homework submittable, being able to post lecture slides and tutorials and having a logout button. For the second sprint, wehave added to feature for students being able to view tutor feedback, lecturers being able to review feedback, lecturers being able to check if students keyed in their attendance using the school WiFi or elsewhere. Lecturers can also see the percentage of total attendance, allow and review a consultation and be able to review the direct messages that students can send to a lecturer.
- Testing / Feedback
  - During the Testing/Feedback phase of the project, our team developed 2 prototypes, Prototype 1 and Prototype 2, to ensure system functionality. Thanks to this, we discovered a bug in our home page after the completion of the first prototype, making us able to fix it when making our second. For the feedback we received, we were able to change thing such as the system layout, visual design, clearer and shorter instructions for easy understanding and usage as well as an overall check to make sure everything's working.
